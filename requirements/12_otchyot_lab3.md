# Отчёт о выполнении лабораторной работы №3

---

## Планирование этапа концептуального проектирования

Работа была организована по этапам, соответствующим методическим рекомендациям лабораторной работы:
1. Выделена основная задача и социально-экономическая цель разработки ProjectFlow.
2. Сформулирована бизнес-постановка и выделены ключевые заинтересованные стороны.
3. Выполнен подробный анализ предметной области с использованием результатов предыдущих лабораторных (lab_1, lab_2).
4. Составлены профили пользователей, ролей, задач, бизнес-процессов.
5. Спроектирована архитектура: охвачены use case, бизнес-процессы, сущности, последовательности событий и интеграции.
6. Проработана база данных (ERD, ORD).
7. Использованы современные методы моделирования: EventStorming, Event Modeling, C4.
8. Все диаграммы реализованы на PlantUML, снабжены аннотированными пояснениями на русском языке.

## Описание выполненных шагов
- Созданы отдельные файлы для постановки задачи и стратегии дизайна.
- Реализован агрегатор/навигатор по всем диаграммам: бизнес-процессов, use case, activity, классовых, object, последовательностей, компонентов, пакетов, развертывания, ERD/ORD.
- Проведено моделирование EventStorming, Event Modeling, подготовлена архитектурная диаграмма C4.
- Все сценарии вариантов использования расписаны по шаблонам Given-When-Then (Gherkin).
- Диаграммы снабжены комментариями, а структура папки полностью соответствует чек-листу задания.
