# Сценарии вариантов использования (Use Case Scenarios) ProjectFlow

## 1. Создание нового проекта
```
Сценарий: Создание нового проекта
Дано: Менеджер заходит в систему ProjectFlow
Когда: Менеджер выбирает опцию «Создать проект», вводит имя, цели и выбирает команду
Тогда: Система сохраняет новый проект и отображает его в списке проектов
```

## 2. Планирование задач
```
Сценарий: Планирование задач в проекте
Дано: Менеджер открыл страницу проекта и доступен backlog
Когда: Менеджер создает задачи, указывает приоритеты и назначает исполнителей
Тогда: Задачи появляются в backlog, исполнителям отправляются уведомления
```

## 3. Выполнение задачи исполнителем
```
Сценарий: Исполнитель берёт задачу в работу
Дано: Исполнитель видит задачи, назначенные на него
Когда: Исполнитель переводит задачу в статус «В работе», пишет комментарии, прикрепляет файлы
Тогда: Система сохраняет изменения и информирует менеджера и команду об обновлении
```

## 4. Аналитика по проекту
```
Сценарий: Просмотр аналитики по проекту
Дано: Руководитель заходит на Dashboard проекта
Когда: Запрашивает сводный отчёт по статусу задач и показателям команды
Тогда: Система отображает актуальные метрики, графики, прогресс-бар по этапам
```

## 5. Администрирование пользователей
```
Сценарий: Администратор добавляет нового пользователя
Дано: Администратор открыт в панели управления пользователями
Когда: Вводит данные нового пользователя и назначает ему роль
Тогда: Пользователь приглашён в систему и получает email с доступом
```
