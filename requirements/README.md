# ProjectFlow: Документация лабораторной работы №3

---

## О проекте

**ProjectFlow** — комплексная система управления проектами и задачами с поддержкой Kanban, Scrum, аналитики, web и мобильной версий. Решение ориентировано на командную работу в IT, учёт метрик, отчётность и профессиональную организацию бизнеса и разработчиков.

---

## Структура папки lab_3

- `specification.md` — постановка задачи, цели, стратегия внедрения, SRS.
- `diagram_usecase.puml` — диаграмма вариантов использования (Use Case), акторы и ключевые действия.
- `usecase_scenarios.md` — подробные сценарии Gherkin (GWT) для всех основных процессов.
- `diagram_activity_*.puml` — диаграммы бизнес-процессов и активностей (планирование, спринт, аналитика).
- `diagram_klass.puml` — главная диаграмма классов сущностей.
- `diagram_object_project.puml` — object diagram (пример связей).
- `diagram_sequence_*.puml` — диаграммы последовательностей (создание задач, синхронизация, управление ролями и др).
- `diagram_component.puml` — диаграмма компонентов системы.
- `diagram_package.puml` — диаграмма пакетов, модульная структура.
- `diagram_deployment.puml` — схема развертывания и инфраструктуры.
- `diagram_erd.puml` — ER-диаграмма БД.
- `diagram_ord.puml` — диаграмма объектно-реляционного отображения.
- `diagram_eventstorming.puml` — спецификация системы по EventStorming.
- `diagram_event_modeling.puml` — спецификация системы по Event Modeling.
- `diagram_c4.puml` — архитектура по методике C4.
