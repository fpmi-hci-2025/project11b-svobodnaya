# GOMS Анализ: Форма управления пользователем
## ProjectFlow - Web приложение

---

## Описание формы

**Форма:** Создание/редактирование пользователя системы  
**Контекст:** Администрирование → Пользователи → "+ Добавить пользователя"  
**Цель:** Добавить нового участника команды с правами доступа

### Поля формы:

1. **Имя** (текстовое, обязательное)
2. **Фамилия** (текстовое, обязательное)
3. **Email** (email input, обязательное, уникальное)
4. **Должность** (текстовое)
5. **Отдел** (dropdown: Development/QA/Design/Management)
6. **Роль в системе** (dropdown: User/Team Lead/Project Manager/Admin)
7. **Проекты** (multi-select: доступные проекты)
8. **Уровень доступа** (radio: Read/Write/Full)
9. **Часовой пояс** (dropdown с поиском)
10. **Язык интерфейса** (dropdown: Русский/English)
11. **Отправить приглашение** (checkbox)
12. Кнопки: **"Создать"**, **"Отмена"**

**Общее количество полей:** 11 + кнопка = 12 действий

## Вариант 1A: Навигация клавиатурой (базовый сценарий)

### Данные для ввода:
- Имя: "Петр" (4 символа)
- Фамилия: "Иванов" (6 символов)
- Email: "p.ivanov@company.com" (21 символ)
- Должность: "Frontend Developer" (18 символов)
- Отдел: Development
- Роль: Team Lead
- Проекты: "Мобильное приложение ЕдаВДом", "Web Dashboard" (2 проекта)
- Уровень доступа: Write
- Часовой пояс: GMT+3 (Minsk)
- Язык: Русский
- Приглашение: Да (отметить)

### Последовательность операторов GOMS:

```
H + M + P + K      // Открыть форму добавления пользователя
M + H              // Обдумывание + переход на клавиатуру
K×4                // Ввод имени "Петр"
K                  // Tab к фамилии
K×6                // Ввод "Иванов"
K                  // Tab к email
M                  // Обдумывание email
K×21               // Ввод email
K                  // Tab к должности
K×18               // Ввод должности
K                  // Tab к отделу
K + K              // Открыть dropdown + выбрать Development (Enter)
K                  // Tab к роли
M                  // Обдумывание роли
K + K + K          // Открыть + стрелка вниз + Enter (Team Lead - 2-й)
K                  // Tab к проектам
M                  // Обдумывание проектов
K                  // Открыть multi-select
K×10 + K           // Поиск "Мобильное" + Enter
K×3 + K            // Поиск "Web" + Enter
K                  // Tab к уровню доступа
K + K              // Стрелка вниз + пробел (выбрать Write)
K                  // Tab к часовому поясу
K + K×5 + K        // Открыть + поиск "GMT+3" + Enter
K                  // Tab к языку
K + K              // Открыть + Enter (Русский первый)
K                  // Tab к чекбоксу приглашения
K                  // Пробел (отметить)
M                  // Финальное обдумывание
K×3                // Tab к кнопке + Enter
```

### Детальный расчет:

**Операторы:**
- H = 0.4 с × 2 = 0.8 с
- P = 1.1 с × 1 = 1.1 с
- K = 0.2 с × (4+6+21+18+10+3+5 текст + 28 служебных) = 0.2 × 95 = 19.0 с
- M = 1.35 с × 5 = 6.75 с

**Общее время:** 0.8 + 1.1 + 19.0 + 6.75 = **27.65 секунд** ≈ **28 секунд**

**Примечание:** Это самая быстрая из трех форм благодаря меньшему объему ввода.

## Вариант 1B: Навигация клавиатурой (с автодополнением)

### Оптимизации:

1. **Email:** Автогенерация из имени и фамилии
   - Было: 21 символ
   - Стало: 2 нажатия (Tab + Enter для подтверждения)
   - Экономия: 19 K = 3.8 с

2. **Должность:** Предложения из справочника
   - Было: 18 символов
   - Стало: 5 нажатий (F + r + o + ↓ + Enter)
   - Экономия: 13 K = 2.6 с

3. **Отдел:** Определяется из должности
   - Было: 3 нажатия
   - Стало: 0 (автоматически Development для Developer)
   - Экономия: 3 K + 1 M = 2.0 с

4. **Проекты:** Последние активные проекты предложены
   - Было: 15 нажатий (поиск обоих)
   - Стало: 4 нажатия (2× Space для выбора из списка)
   - Экономия: 11 K = 2.2 с

5. **Часовой пояс:** Определяется автоматически
   - Было: 8 нажатий
   - Стало: 0 (из настроек организации)
   - Экономия: 8 K = 1.6 с

6. **Язык:** По умолчанию язык браузера
   - Было: 3 нажатия
   - Стало: 0 (автоматически)
   - Экономия: 3 K = 0.6 с

**Общая экономия:** 57 K + 1 M = 12.8 секунд

**Новое время:** 27.65 - 12.8 = **14.85 секунд** ≈ **15 секунд**

**Улучшение:** 46% быстрее базового варианта

## Вариант 2A: Навигация мышью (базовый сценарий)

### Последовательность операторов:

```
H + M + P + K          // Открыть форму
M + P + K              // Клик на поле "Имя"
H + K×4                // Ввод имени
H + P + K              // Клик на "Фамилия"
H + K×6                // Ввод фамилии
H + P + K              // Клик на "Email"
H + M + K×21           // Ввод email
H + P + K              // Клик на "Должность"
H + K×18               // Ввод должности
H + P + K              // Клик на dropdown "Отдел"
P + K                  // Клик на Development
H + M + P + K          // Клик на dropdown "Роль"
P + K                  // Клик на Team Lead
H + P + K              // Клик на multi-select "Проекты"
H + K×10               // Поиск первого проекта
H + P + K              // Клик на результат
H + K×3                // Поиск второго
H + P + K              // Клик на результат
P + K                  // Клик на radio "Write"
P + K                  // Клик на dropdown "Часовой пояс"
H + K×5                // Поиск
H + P + K              // Клик на результат
P + K                  // Клик на dropdown "Язык"
P + K                  // Клик на Русский
P + K                  // Клик на checkbox приглашения
M + P + K              // Клик "Создать"
```

### Детальный расчет:

**Операторы:**
- H = 0.4 с × 15 = 6.0 с
- P = 1.1 с × 23 = 25.3 с
- K = 0.2 с × (4+6+21+18+10+3+5 текст + 23 клика) = 0.2 × 90 = 18.0 с
- M = 1.35 с × 5 = 6.75 с

**Общее время:** 6.0 + 25.3 + 18.0 + 6.75 = **56.05 секунд** ≈ **56 секунд**

**Сравнение:** На 100% медленнее варианта 1A (в 2 раза!)

## Вариант 2B: Навигация мышью (с автодополнением)

Применяются те же оптимизации, что и в 1B:
- Автогенерация email
- Справочник должностей
- Автоопределение отдела
- Последние проекты
- Автоопределение часового пояса и языка

**Экономия операций:**
- Email: -19 K + переключения = -4.5 с
- Должность: -13 K + переключения = -3.2 с
- Отдел: -3 клика P+K = -3.3 с
- Проекты: -11 K + переключения = -3 с
- Часовой пояс: -8 K + переключения = -2.5 с
- Язык: -3 клика = -3.3 с

**Общая экономия:** ≈20 секунд

**Новое время:** 56.05 - 20 = **36.05 секунд** ≈ **36 секунд**

**Улучшение:** 36% быстрее варианта 2A

## Анализ узких мест

### Проблемные зоны:

1. **Ввод email вручную (2A)**
   - 21 символ = 4.2 секунды
   - **Решение:** Автогенерация из имени.фамилия@company.com

2. **Ввод должности вручную**
   - 18 символов = 3.6 секунды
   - **Решение:** Справочник типовых должностей

3. **Поиск проектов для назначения**
   - 13 символов = 2.6 секунды
   - **Решение:** Топ-5 активных проектов сразу видны

4. **Множественные переключения мышь↔клавиатура (2A)**
   - 15 переключений = 6 секунд потерь
   - **Решение:** Полная клавиатурная навигация

## Валидация и обработка ошибок

### Критичные проверки:

1. **Email уникальность**
   - Проверка в реальном времени (после потери фокуса)
   - "Пользователь с таким email уже существует"

2. **Email формат**
   - Валидация RFC 5322
   - Проверка MX записи домена (опционально)

3. **Обязательные поля**
   - Имя, Фамилия, Email, Роль - обязательны
   - Остальные можно заполнить позже

4. **Права доступа**
   - Admin может назначать Admin
   - PM не может назначать Admin
   - Подсказка о правах при выборе роли
