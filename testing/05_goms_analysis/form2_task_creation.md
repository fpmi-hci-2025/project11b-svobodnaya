# GOMS Анализ: Форма создания задачи
## ProjectFlow - Web приложение

---

## Описание формы

**Форма:** Создание новой задачи в проекте  
**Контекст:** Проект → Кнопка "+ Создать задачу"  
**Цель:** Быстро создать задачу с основными параметрами

### Поля формы:

1. **Название задачи** (текстовое, обязательное)
2. **Тип задачи** (dropdown: User Story/Task/Bug/Epic)
3. **Приоритет** (dropdown: Low/Medium/High/Critical)
4. **Story Points** (number input: 1-13)
5. **Описание** (textarea с форматированием)
6. **Исполнитель** (dropdown с поиском)
7. **Спринт** (dropdown: Backlog/Sprint 1/Sprint 2...)
8. **Срок** (datepicker)
9. **Метки** (multi-select tags: frontend, backend, mobile...)
10. Кнопки: **"Создать"**, **"Создать и добавить еще"**, **"Отмена"**

**Общее количество полей:** 9 + кнопка = 10 действий

## Вариант 1A: Навигация клавиатурой (базовый сценарий)

### Данные для ввода:
- Название: "Разработка экрана авторизации" (30 символов)
- Тип: User Story
- Приоритет: High
- Story Points: 5
- Описание: "Реализовать вход через email и пароль..." (80 символов)
- Исполнитель: "Иванов Петр" (11 символов для поиска)
- Спринт: Sprint 1
- Срок: 10.12.2024 (10 символов)
- Метки: frontend, mobile, authentication (3 тега)

### Последовательность операторов GOMS:

```
H + M + P + K  // Открыть форму создания задачи
M + H          // Обдумывание + переход на клавиатуру
K×30           // Ввод названия
K              // Tab к полю "Тип"
K + K          // Открыть dropdown (Enter) + выбрать (↓ Enter)
K              // Tab к полю "Приоритет"
M              // Обдумывание приоритета
K + K + K + K  // Открыть + ↓ ↓ Enter (High - 3-й пункт)
K              // Tab к Story Points
K×1            // Ввод "5"
K              // Tab к описанию
M              // Обдумывание описания
K×80           // Ввод описания
K              // Tab к исполнителю
M              // Обдумывание кого назначить
K + K×11 + K   // Открыть + поиск + Enter
K              // Tab к спринту
K + K          // Открыть + выбрать Sprint 1
K              // Tab к сроку
K×10           // Ввод даты
K              // Tab к меткам
K×9 + K        // "frontend" + Enter
K×6 + K        // "mobile" + Enter  
K×14 + K       // "authentication" + Enter
M              // Финальное обдумывание
K×2            // Tab к кнопке + Enter
```

### Детальный расчет:

**Операторы:**
- H = 0.4 с × 2 = 0.8 с
- P = 1.1 с × 1 = 1.1 с
- K = 0.2 с × (30 + 80 + 11 + 10 + 9 + 6 + 14 + 28 служебных) = 0.2 × 188 = 37.6 с
- M = 1.35 с × 5 = 6.75 с

**Общее время:** 0.8 + 1.1 + 37.6 + 6.75 = **46.25 секунд** ≈ **46 секунд**

## Вариант 1B: Навигация клавиатурой (с автодополнением)

### Оптимизации:
1. **Описание:** Использование шаблона для User Story
   - Было: 80 символов
   - Стало: 3 нажатия (↓ + Enter для выбора шаблона + подстановка)
   - Экономия: 77 K = 15.4 с

2. **Исполнитель:** Последний назначенный участник
   - Было: 13 нажатий (открыть + поиск + enter)
   - Стало: 2 нажатия (открыть + enter на первом)
   - Экономия: 11 K = 2.2 с

3. **Метки:** Частые метки предложены
   - Было: 29 нажатий (печать всех тегов)
   - Стало: 6 нажатий (3× выбор из списка Enter)
   - Экономия: 23 K = 4.6 с

4. **Спринт:** Текущий спринт по умолчанию
   - Было: 3 нажатия (Tab + открыть + выбрать)
   - Стало: 1 нажатие (Tab, уже выбран)
   - Экономия: 2 K = 0.4 с

**Общая экономия:** 113 K = 22.6 секунд

**Новое время:** 46.25 - 22.6 = **23.65 секунд** ≈ **24 секунды**

**Улучшение:** 48% быстрее базового варианта

## Вариант 2A: Навигация мышью (базовый сценарий)

### Последовательность операторов:

```
H + M + P + K  // Открыть форму
M + P + K      // Клик на поле "Название"
H + K×30       // Переход на клавиатуру + ввод
H + P + K      // Клик на dropdown "Тип"
P + K          // Клик на User Story
M              // Обдумывание
P + K          // Клик на dropdown "Приоритет"
P + K          // Клик на High
P + K          // Клик на Story Points
H + K×1        // Ввод 5
H + P + K      // Клик на "Описание"
H + M + K×80   // Ввод описания
H + P + K      // Клик на "Исполнитель"
H + K×11       // Поиск
H + P + K      // Клик на результат
P + K          // Клик на "Спринт"
P + K          // Клик на Sprint 1
P + K          // Клик на датапикер
H + K×10       // Ввод даты
H + P + K      // Клик на метки
H + K×9        // frontend
H + P + K      // выбор из списка
H + K×6        // mobile
H + P + K      // выбор
H + K×14       // authentication
H + P + K      // выбор
M + P + K      // Клик "Создать"
```

### Детальный расчет:

**Операторы:**
- H = 0.4 с × 18 = 7.2 с
- P = 1.1 с × 19 = 20.9 с
- K = 0.2 с × (30 + 80 + 11 + 10 + 9 + 6 + 14 + 19 кликов) = 0.2 × 179 = 35.8 с
- M = 1.35 с × 5 = 6.75 с

**Общее время:** 7.2 + 20.9 + 35.8 + 6.75 = **70.65 секунд** ≈ **71 секунда**

**Сравнение:** На 54% медленнее варианта 1A

## Вариант 2B: Навигация мышью (с автодополнением)

Применяются те же оптимизации, что и в 1B:
- Шаблон для описания
- Последний исполнитель
- Частые метки
- Текущий спринт

**Экономия операций:**

**Общая экономия:** ≈27 секунд

**Новое время:** 70.65 - 27 = **43.65 секунд** ≈ **44 секунды**

**Улучшение:** 38% быстрее варианта 2A

## Анализ узких мест

### Проблемные зоны:

1. **Ввод описания (80 символов)**
   - Занимает: 16 секунд (35% времени в 1A)
   - **Решение:** Шаблоны User Story экономят 15.4 с

2. **Переключения рука↔мышь (вариант 2A)**
   - 18 переключений × 0.4с = 7.2 секунды потерь
   - **Решение:** Клавиатурная навигация

3. **Перемещение мыши (вариант 2A)**
   - 19 перемещений × 1.1с = 20.9 секунд
   - **Решение:** Крупные таргеты, логичная последовательность

4. **Ввод меток вручную**
   - 29 символов = 5.8 секунд
   - **Решение:** Частые метки, автодополнение

## Рекомендации по оптимизации

### Высокий приоритет (реализовать обязательно):

1. ✅ **Шаблоны для типов задач**
   - User Story: "Как [роль], я хочу [действие], чтобы [цель]"
   - Bug: "При [условие] происходит [проблема]"
   - Task: Свободная форма
   - **Экономия: 15+ секунд (32%)**

2. ✅ **Smart defaults**
   - Последний исполнитель
   - Текущий спринт
   - Приоритет Medium по умолчанию
   - **Экономия: 3-4 секунды (6-8%)**

3. ✅ **Частые метки**
   - Топ-10 меток проекта всегда видны
   - Клик вместо печати
   - **Экономия: 4-6 секунд (8-13%)**

### Средний приоритет:

4. ✅ **Быстрое создание (Quick Add)**
   - Только название + создать
   - Остальное заполнить потом
   - **Цель: < 10 секунд**

5. ✅ **Копирование похожей задачи**
   - "Создать похожую" на существующей задаче
   - Переносит все кроме названия

6. ✅ **Полная клавиатурная навигация**
   - Все dropdowns работают со стрелками
   - Enter для выбора
   - **Текущий статус: ✅ Реализовано**

### Низкий приоритет:

7. ✅ **Горячие клавиши для приоритета**
   - Ctrl+1/2/3/4 для Low/Med/High/Critical
   - **Экономия: 1-2 секунды**

8. ✅ **Голосовой ввод описания**
   - Для длинных описаний
   - **Экспериментально**
