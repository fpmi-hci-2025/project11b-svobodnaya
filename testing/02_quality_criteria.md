## 1. Общие целевые показатели

### 1.1 Метрики успешности (Task Success)

**Определение:** Процент участников, которые полностью и корректно выполнили задание без критичных ошибок.

**Целевые значения по приоритету:**

| Приоритет задания | Целевая успешность | Минимум приемлемый |
|-------------------|--------------------|--------------------|
| Критичный | ≥ 90% | 80% |
| Высокий | ≥ 85% | 75% |
| Средний | ≥ 80% | 70% |
| Низкий | ≥ 75% | 65% |

**Критичные задания ProjectFlow:**
- Создание проекта: ≥90%
- Создание задачи: ≥90%
- Изменение статуса: ≥90%

### 1.2 Метрики времени (Time-on-Task)

**Определение:** Время, затраченное участником на выполнение задания от начала до успешного завершения.

**Целевые значения:**

| Сложность | Опытный пользователь | Средний пользователь | Новичок |
|-----------|----------------------|----------------------|---------|
| Простой | < 2 мин | 2-3 мин | 3-4 мин |
| Средний | 2-4 мин | 4-6 мин | 6-8 мин |
| Сложный | 4-6 мин | 6-10 мин | 10-15 мин |

**Критерий приемлемости:** 
- ≥75% участников укладываются в целевое время для их уровня опыта

### 1.3 Метрики ошибок (Error Rate)

**Типы ошибок:**
- **Критичная:** Невозможность завершить задание, потеря данных
- **Существенная:** Неверный путь, требующий отката действий
- **Незначительная:** Временная неуверенность, исправленная самостоятельно

**Целевые значения:**

| Тип ошибки | Максимум на задание | Максимум на сессию |
|------------|---------------------|---------------------|
| Критичная | 0 | 0 |
| Существенная | ≤ 1 | ≤ 3 |
| Незначительная | ≤ 2 | ≤ 5 |

### 1.4 Метрики эффективности (Efficiency)

**Определение:** Отношение минимально необходимых действий к фактически выполненным.

```
Efficiency = (Минимум шагов / Фактические шаги) × 100%
```

**Целевые значения:**
- Оптимальный путь: ≥90% efficiency
- Приемлемый: 75-89%
- Неэффективный: <75%

## 8. Комбинированные показатели

### 8.1 Единый показатель usability (SUM)

```
SUM = (Completion + Satisfaction + (1 - Time%) + (1 - Errors%)) / 4
```

**Целевое значение:** ≥0.80 (80%)

Где:
- Completion = % успешных заданий
- Satisfaction = нормализованная SUS оценка (0-1)
- Time% = процент превышения целевого времени
- Errors% = процент от максимально допустимых ошибок

## 9. Критерии приоритизации проблем

### 9.1 Серьезность проблем

**Критичная (Priority 1):**
- Блокирует выполнение критичного задания
- Успешность <70% для критичного функционала
- Потеря данных
- **Действие:** Исправить немедленно

**Высокая (Priority 2):**
- Снижает эффективность >30%
- Успешность 70-80% для важного функционала
- Частые ошибки (>50% пользователей)
- **Действие:** Исправить в следующей итерации

**Средняя (Priority 3):**
- Снижает эффективность 15-30%
- Успешность 80-90%
- Неудобство для части пользователей
- **Действие:** Запланировать улучшение

**Низкая (Priority 4):**
- Косметические недочеты
- Не влияет на выполнение задач
- **Действие:** Backlog для будущих версий

### 9.2 Формула приоритета

```
Priority Score = Severity × Frequency × Impact

Где:
- Severity (1-5): серьезность проблемы
- Frequency (1-5): как часто встречается
- Impact (1-5): влияние на пользовательский опыт
```

**Распределение:**
- 75-125: Критичная (P1)
- 40-74: Высокая (P2)
- 15-39: Средняя (P3)
- 1-14: Низкая (P4)
